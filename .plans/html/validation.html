Evidence plan

Ship only when trust metrics say yes.

The MVP should be judged by safety, continuation success, restore reliability, and whether developers understand what Devbox is doing to their code folder.

Hard stop: any confirmed unrecoverable data loss, Git corruption, or secret leak blocks expansion until root cause and regression tests exist.

North Star

Successful Workspace Continuations

Cross-device resume sessions where the folder is ready, required files are available, no blocking conflict exists, and the user edits, runs, tests, commits, snapshots, or restores within 30 minutes.

Instrumentation

Area Events Decision

Experiment Gates

Phase 0

Local restore trust

10 users restore real WIP in non-critical repos and describe it as safer than stash/WIP commits.

Phase 1

Second-device magic

25 alpha users complete desktop-to-laptop continuation with no manual file movement.

Phase 2

Paid beta readiness

500 beta users, 40% second-device activation, 30% weekly active synced folders.

Launch Readiness

  • Critical incident runbook exists and has been rehearsed.
  • Regression suite covers every known restore, Git, watcher, secret, and conflict bug.
  • Support bundle excludes source by default and redacts paths when requested.
  • R2 operation cost, local cache amplification, and storage per active user are visible.
  • Every user-visible warning has a next action.
  • The product can export or remove a folder cleanly.
  • `scripts/alpha-two-device-smoke.sh` passes and writes redacted evidence logs.
  • Alpha tool archives contain `devbox`, `devbox-daemon`, `devbox-metadata`, docs, env template, and helper scripts.
  • The desktop control surface builds, passes safety scan, and does not imply sync is running until daemon IPC exists.
The MVP passes when developers let it touch real work twice.
