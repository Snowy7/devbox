Overview Phase 1 Phase 2 Architecture Validation
4 to 6 weeks

Make local snapshots boringly safe.

Before cloud sync, prove Devbox can analyze developer folders, classify files, snapshot WIP, restore safely, detect local changes, and avoid corrupting Git repos.

Exit gate: snapshot and restore 20 representative repos with no Git corruption, uncommitted work preserved, and generated directories suppressed by policy.

Work Areas

Scanner and Classifier

Detect code roots, nested folders, Git repos, stack signals, generated directories, and secret candidates.

Snapshot Engine

Create manifests, hash chunks with BLAKE3, store content-addressed files, and record pending local change operations in SQLite.

Restore Engine

Restore file, folder, and code-root state atomically with rollback and pre-restore safety checks.

Deliverables

  • Rust workspace with daemon crate, CLI crate, core library, and integration test harness.
  • SQLite schema for folder scopes, snapshots, manifests, chunks, operations, policies, and restore attempts.
  • Default dev policy for `node_modules`, `.next`, `dist`, `target`, `.venv`, `.cache`, and common build outputs.
  • Git detector that records repo root, HEAD, dirty state, ignored state, remotes, submodules, and worktree shape.
  • CLI commands: `devbox scan`, `devbox snapshot`, `devbox status`, `devbox restore`, `devbox explain`.
  • Local change feed: compare the current workspace with the latest snapshot and persist pending file operations without cloud upload.
  • Destructive test matrix for restore, interrupted writes, symlinks, permissions, case conflicts, and Git status.

Week Shape

Week Focus Proof
1 Repo layout, scanner, policy skeleton, SQLite migrations. Can scan a code root and classify developer folders.
2 Manifest model, hashing, local object cache. Can create deterministic snapshots.
3 Restore engine, atomic materialization, rollback. Can restore modified and untracked files.
4 Git safety, generated artifact suppression, secret block prompts. Git status remains valid after restore.
5-6 Local change feed, test hardening, and alpha demo path. 20 representative repos pass the safety matrix, and pending operations are repeatable.
Phase 0 is not a toy. It is the trust foundation for everything after it.
