8 to 10 weeks

Prove desktop-to-laptop continuation.

Take the local snapshot foundation through a local watcher first, then add encrypted sync, device pairing, second-device materialization, and a minimal control surface for trusted alpha users.

Exit gate: 25 alpha users complete the workflow with P50 project-ready time under 10 minutes and zero data loss or Git corruption incidents.

Deliverables

Sync Backend

  • Rust API with device pairing and auth.
  • Hosted metadata API foundation for accounts, devices, projects, published manifests, and compare-and-set cursors. SQLite-backed dev/test storage is complete; publish/import/materialize can opt into in-process mock-dev metadata wiring; account-session auth is now modeled for hosted handlers; Postgres remains the production boundary.
  • R2 blob backend behind a provider interface.
  • Encrypted immutable blob upload and download. Local filesystem, S3-compatible R2/S3/MinIO, and hosted object-transfer provider foundations are complete; managed credential lease metadata and hosted shared-bucket prefix grants keep external tester clients away from raw bucket keys, while live cloud provisioning remains deferred.

Client Sync

  • Local watcher feeds the pending operation log after debounced filesystem events.
  • Upload queue, retry, resume, and backoff.
  • Local/mock publish, import, and materialize foundation through encrypted local remote.
  • Live daemon sync loop for deterministic scan/publish and latest-remote import/materialize with cursor preflight.
  • Download and materialization queue.
  • Local device cursor reconciliation before import/materialization.
  • Conflict-as-divergent-snapshot refusal records for divergent local/mock sync.

CLI and Minimal UI

  • `devbox init`, `devices`, `sync`, `pause`, `restore`.
  • Env-backed Electron shell for alpha onboarding, status, hosted config, object-access grants, pairing, and live sync commands.
  • Tray state: ready, needs-config, command-only, blocked.
  • Redacted control surface for local paths, projects, conflicts, devices, secrets, remote prefix state, and settings.
  • macOS/Linux alpha release scripts for CLI/daemon/metadata tools and unsigned desktop bundles.
  • Deterministic two-device smoke harness with redacted evidence logs.
  • Support bundle without source content by default.

Safety

  • Local high-confidence secret blocking before uploadable blobs.
  • Explicit local secret block/template/envelope-reference policy records.
  • Git verification after materialization.
  • Atomic writes with rollback.
  • Manual pause and project exclusion.

Current Foundation

  • Local watcher feeds the pending operation log.
  • Local account and current-device identity initialize idempotently in SQLite.
  • The account/device model remains multi-device-capable even though one installation has one current local device.
  • Encrypted blob upload and download work through a local filesystem remote provider without cloud credentials.
  • S3-compatible encrypted blob transport can target Cloudflare R2, AWS S3, or MinIO with redacted environment-based credential configuration.
  • Hosted metadata API/store/handler foundations model accounts, devices, projects, published snapshot manifests, and server-side compare-and-set device/project cursors.
  • Publish/import/materialize can opt into in-process mock-dev hosted metadata for manifest discovery and server-side cursor compare-and-set.
  • Live daemon sync can scan/debounce, persist idempotent snapshots, publish encrypted objects, register hosted mock-dev metadata, discover latest remote snapshots, and import/materialize with conflict refusal.
  • Local/mock auth sessions, pairing invitations, approved-device trust records, key envelopes, revocation markers, and local device/project cursors are in place.
  • Production-shaped account ownership proof and account session primitives model provider subject/email/domain proof, token-hash sessions, expiration, revocation, and no-network CLI/dev persistence.
  • Hosted metadata handlers now support explicit mock-dev headers for tests/dev plus production-shaped account-session bearer auth resolved through the hosted session store, with account-scoped device/project/snapshot/cursor operations.
  • Hosted metadata can now run as a single-instance alpha API with `/ready`, one-time invite login, bearer session status/logout, and mock-dev auth disabled by default in the server binary.
  • Managed object credential lease and object-access grant primitives model account/session/project-scoped R2/S3/MinIO-compatible provider references, canonical shared-bucket prefixes, capabilities, expiration, revocation, and rotation generation without live provider provisioning.
  • Hosted object transfer now proxies encrypted object put/get/head/list through the metadata API with account-session auth and server-held bucket credentials.
  • Production-shaped pairing recovery and rotation primitives model receiver-generated join/complete handoff, recovery grant references, revocation, device rotation intents, and key-envelope rotation generation with explicit secret-bearing alpha payloads.
  • Local/mock second-device materialization now proves encrypted publish/import, paired-receiver decrypt without publisher DB access, deferred materialize cursor commit, and safe restore-engine apply.
  • Local high-confidence secret detection blocks detected files before blob-cache writes and uploadable local/mock sync blobs.
  • Local conflict-as-divergent-snapshot compare and persistence metadata records readable path-level divergence without source bytes.
  • Local sync preflight refuses divergent import/materialization before receiver file blobs are downloaded or applied, and leaves cursors unchanged when conflicts are created.
  • Env-backed redacted Electron alpha shell, explicit path-scoped secret policy records, guarded manual conflict resolution records, alpha package scripts, and deterministic two-device smoke evidence are now in place.
  • OAuth/OIDC login, signed installers, live Cloudflare/AWS object credential provisioning, production pairing/recovery UX, automatic merge/apply resolution, paid/team/agent/Git replacement work, and deployment hardening remain deferred.
