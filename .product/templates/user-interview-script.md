# User Interview Script

## Opening

Thanks for talking with us. We are exploring a developer-native way to make your code folder and work-in-progress follow you across machines. We are not testing whether you like our pitch. We are trying to understand where your current workflow breaks.

## Context

1. What kind of development work do you do?
2. How many machines do you code on weekly?
3. What operating systems do you use?
4. What IDEs and terminals do you use?
5. Where do your projects live locally?

## Current Workflow

1. Walk me through switching from one machine to another.
2. What usually works?
3. What usually breaks?
4. How do you handle uncommitted work?
5. Do you ever avoid switching machines because of project state?

## Existing Solutions

1. Have you tried Dropbox, iCloud, OneDrive, Syncthing, rsync, Git, dotfiles, dev containers, or Codespaces for this?
2. What did you keep using?
3. What did you abandon?
4. What felt unsafe?

## Trust

1. What would make you afraid to let a tool touch your code folder?
2. Which files should never sync automatically?
3. What would you need to see before using this on real work?
4. What restore guarantee would matter?

## Value

1. If this worked perfectly, how often would you use it?
2. Would this be personal, team, or company-paid?
3. Would USD 10/month feel cheap, fair, expensive, or absurd?
4. What would make this a must-have?

## Close

Would you be willing to test this on a non-critical repo when the prototype is ready?
